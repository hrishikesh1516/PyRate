## What is the preservation (sampling) process?

In paleontology, **only a fraction of species that ever lived leave fossils that are both preserved and discovered**. The probability that a lineage is sampled (i.e., leaves at least one fossil occurrence per time interval) is governed by the **preservation (or sampling) rate**, usually denoted **q** (occurrences per lineage per million years, My).

PyRate models the appearance of fossil occurrences as a **Poisson process** — the same mathematical framework used to model random events arriving at a certain rate. Different models capture different biological and taphonomic realities:

| Scenario | Recommended model |
|---|---|
| Preservation is roughly constant across the clade's history | HPP |
| Preservation peaks near the middle of each lineage's lifespan | NHPP (default) |
| Preservation varies across geological epochs (e.g., due to sea-level, rock record biases) | TPP |
| Some lineages are inherently more preservable than others (body size, habitat, mineralogy) | any of the above + Gamma (`-mG`) |

Getting the preservation model right matters because **biased or misspecified preservation rates can inflate or deflate inferred speciation and extinction rates**.

***

## Input data requirements

PyRate requires a **Python input file** (`.py`) with per-lineage arrays of fossil occurrence ages (in Ma). This file is typically generated by the R utility function `extract.ages()` or `extract.ages.pbdb()` provided in `pyrate_utilities.r`. See [Tutorial #1](https://github.com/dsilvestro/PyRate/blob/master/tutorials/pyrate_tutorial_1.md) for instructions on generating this input file.

### Key requirements for preservation analyses

1. **Multiple occurrences per lineage** — at least 3–5 occurrences per taxon help constrain preservation rates. With only 1–2 occurrences per taxon, rate estimates will be very uncertain.

2. **Age uncertainty** — PyRate accounts for age uncertainty by analysing multiple replicated datasets, each with ages independently resampled from their stratigraphic ranges. Generate replicates using `replicates=10` (or more) in `extract.ages()`.

3. **Representative taxonomic sampling** — The dataset should cover all (or a representative sample of) lineages in the clade. Systematic inclusion of only well-preserved taxa will bias preservation rate estimates upward.

4. **Extant taxa flagged correctly** — Species still alive today must be declared as "extant" (Status column = "extant") so PyRate can correctly account for their non-zero survival probability.

### Minimum input file structure

The source table (e.g. `Felidae_preservation_example.txt`) must contain at least four tab-separated columns:

| Species | Status | min\_age | max\_age |
|---|:---:|---:|---:|
Proailurus\_lemanensis | extinct | 20.43 | 23.03
Pseudaelurus\_quadridentatus | extinct | 11.63 | 17.0
Panthera\_leo | extant | 0.01 | 0.7

Ages are in **Ma (millions of years ago)**. The `min_age` and `max_age` columns represent the stratigraphic range of each **occurrence** (one row per fossil find, not per species).

Convert this table to a PyRate input file with R:

```r
source(".../PyRate/pyrate_utilities.r")
extant_felids <- c("Panthera_leo", "Panthera_tigris", "Acinonyx_jubatus")
extract.ages(file = ".../PyRate/example_files/Felidae_preservation_example.txt",
             extant_species = extant_felids,
             replicates = 10)
```

This generates `Felidae_preservation_example_PyRate.py` (with 10 replicated age samples) and `Felidae_preservation_example_TaxonList.txt`.

A **ready-to-use single-replicate input file** is also provided at:  
`example_files/Felidae_preservation_PyRate.py`

## Preservation models in PyRate

### 1. Non-homogeneous Poisson process (NHPP) — default

The **NHPP** model assumes that within each lineage's lifespan, preservation rates change through time following a **bell-shaped (logistic) curve**. Rates peak near the middle of the lineage's temporal range and are lower near origination and extinction. This reflects the observation that many clades accumulate more records through time and then lose them near extinction.

**This is PyRate's default model** — no additional flag is needed:

```bash
python PyRate.py .../Felidae_preservation_PyRate.py -A 4
```

**When to use NHPP:**
- When you have no strong *a priori* reason to assume constant preservation.
- As a default starting model.

**Limitation:** The bell-shaped assumption may not match all clades, and the model estimates fewer preservation parameters than TPP (see below).

---

### 2. Homogeneous Poisson process (HPP)

The **HPP** model assumes that preservation rate **q** is constant across all lineages and all time intervals. Only one preservation parameter is estimated.

```bash
python PyRate.py .../Felidae_preservation_PyRate.py -mHPP -A 4
```

**When to use HPP:**
- When the dataset is small and you cannot reliably estimate additional parameters.
- As a null model to compare against more complex models.

**Output:** The `mcmc.log` file contains a single column `q_rate` representing the constant preservation rate.

---

### 3. Time-variable Poisson process (TPP)

The **TPP** model estimates a separate preservation rate for each predefined time interval (e.g., geological epochs). This is the most realistic model when sampling intensity varies through time — for example, due to changes in rock record completeness, sampling effort, or environmental conditions.

You must provide a file listing the **boundaries** between time intervals. An example (Cenozoic + Mesozoic epochs) is provided in `example_files/epochs_q.txt`. A shorter file covering only the Neogene–Quaternary is appropriate for younger datasets.

```bash
python PyRate.py .../Felidae_preservation_PyRate.py \
    -qShift .../example_files/epochs_q.txt -A 4
```

**Controlling the prior on preservation rates:**  
The default prior on each interval's q is a Gamma distribution with shape = 1.5 and rate = 1.5. You can change this with `-pP shape rate`:

```bash
# More diffuse prior (shape=1.5, rate=0 estimates rate from the data)
python PyRate.py .../Felidae_preservation_PyRate.py \
    -qShift .../example_files/epochs_q.txt -pP 1.5 0 -A 4
```

Setting the rate to **0** lets PyRate estimate a hyper-prior on the rate parameter from the data — **recommended whenever multiple preservation rates are estimated**.

**Output:** The `mcmc.log` file contains columns `q_0, q_1, ..., q_n` (from **oldest to youngest** interval).

---

### 4. Gamma model of rate heterogeneity (mG)

The **Gamma model** adds **among-lineage rate heterogeneity**: some species are inherently easier to find as fossils (large body size, hard parts, marine habitat) while others are rarely preserved. This is modelled by assuming each lineage's preservation rate is drawn from a Gamma distribution with shape parameter **alpha** (α).

- **High α** → low heterogeneity (all lineages have similar q)
- **Low α** → high heterogeneity (preservation spans orders of magnitude)

The Gamma model can be combined with **any** base preservation model:

```bash
# NHPP + Gamma
python PyRate.py .../Felidae_preservation_PyRate.py -mG -A 4

# HPP + Gamma
python PyRate.py .../Felidae_preservation_PyRate.py -mHPP -mG -A 4

# TPP + Gamma
python PyRate.py .../Felidae_preservation_PyRate.py \
    -qShift .../example_files/epochs_q.txt -mG -A 4
```

**When to use `-mG`:**  
Add the Gamma model to whichever base model is selected by model testing (see below). It is almost always beneficial when the dataset contains taxa from diverse habitats, body sizes, or preservational environments.

**Output:** An additional column `alpha` is added to `mcmc.log`.

---

### 5. Combining TPP with Gamma heterogeneity

The most flexible setup combines time-variable rates (TPP) with among-lineage heterogeneity (Gamma). You can also log the **per-lineage relative preservation rate** with `-log_sp_q_rates`:

```bash
python PyRate.py .../Felidae_preservation_PyRate.py \
    -qShift .../example_files/epochs_q.txt -mG -log_sp_q_rates -A 4
```

This adds a log file (`*_sp_q_rates.log`) containing the posterior distribution of the relative preservation rate for each lineage — useful to identify which taxa are systematically over- or under-sampled.

## Choosing and testing preservation models

### Automated model testing with `-PPmodeltest`

PyRate implements a **maximum-likelihood test** to compare NHPP, HPP, and TPP preservation models using likelihood-ratio statistics. This should be run **before** the full MCMC analysis to select the best base model:

```bash
python PyRate.py .../Felidae_preservation_PyRate.py \
    -qShift .../example_files/epochs_q.txt -PPmodeltest
```

The output is printed to the screen and shows log-likelihoods and p-values for each model comparison. **Select the model with the best (highest) log-likelihood that is significantly better than simpler models.**

> **Note:** The Gamma model (`-mG`) is **not** part of this test; add it to the winning base model as a standard practice.

### Recommended workflow for model selection

1. Run `-PPmodeltest` to compare NHPP, HPP, and TPP.
2. Adopt the best-supported base model.
3. Add `-mG` to the selected base model.
4. If the full MCMC run was already performed, re-run with the chosen model and compare posterior predictive fit.

## Interpreting preservation parameters

### The preservation rate q

The parameter `q_rate` (or `q_0, q_1, ...` for TPP) represents the **mean number of fossil occurrences per lineage per million years**.

| q value | Interpretation |
|---|---|
| < 0.1 | Very sparse fossil record — most time intervals contain no fossils |
| 0.1 – 0.5 | Moderate record — typical for many macrofossil groups |
| 0.5 – 2.0 | Good record — each lineage leaves ~1 occurrence per epoch |
| > 2.0 | Excellent record — multiple occurrences per million years per lineage |

**In practice:** Preservation rates often vary by one to two orders of magnitude across geological time and between clades. Intervals with high sea-level (marine transgressions), humid climates, or intensive collection effort tend to have higher q.

### The Gamma shape parameter alpha

The column `alpha` in `mcmc.log` is the shape of the Gamma distribution controlling among-lineage heterogeneity.

| α value | Interpretation |
|---|---|
| < 0.5 | Extreme heterogeneity — a few taxa dominate the record |
| 0.5 – 2.0 | Moderate heterogeneity — typical for diverse clades |
| > 5.0 | Low heterogeneity — most lineages have similar preservation rates |

A **low α with wide 95% credible interval** suggests the data cannot resolve heterogeneity well — this is common with small datasets and should be interpreted cautiously.

### Reading preservation parameters from mcmc.log

Open the `mcmc.log` file in [Tracer](https://github.com/beast-dev/tracer/releases) or in R:

```r
log <- read.table("pyrate_mcmc_logs/Felidae_preservation_1_mcmc.log",
                  header = TRUE, sep = "\t")

# Discard first 20% as burn-in
burnin <- ceiling(nrow(log) * 0.2)
post   <- log[(burnin + 1):nrow(log), ]

# Summarize preservation rate
cat("Mean q_rate:", mean(post$q_rate), "\n")
cat("95% CI:", quantile(post$q_rate, c(0.025, 0.975)), "\n")

# Summarize Gamma shape (if -mG was used)
if ("alpha" %in% names(post)) {
  cat("Mean alpha:", mean(post$alpha), "\n")
  cat("95% CI alpha:", quantile(post$alpha, c(0.025, 0.975)), "\n")
}
```

## Preservation rates through time

### Plotting TPP rates with `-plotQ`

After a TPP run, preservation rates through time can be summarized and plotted directly from PyRate:

```bash
python PyRate.py -plotQ \
    pyrate_mcmc_logs/Felidae_preservation_1_mcmc.log \
    -qShift example_files/epochs_q.txt \
    -b 200
```

The `-b 200` flag discards the first 200 samples as burn-in (adjust based on your chain length and sampling frequency). This generates an R script and a PDF plot showing mean preservation rates and 95% credible intervals for each epoch.

### Plotting with the provided R helper script

A ready-to-use R script is provided in `example_files/plot_preservation_rates.R`. It can handle HPP, NHPP, and TPP output and produces a multi-panel PDF:

```bash
# HPP or NHPP — only plots q_rate and alpha posteriors
Rscript example_files/plot_preservation_rates.R \
    pyrate_mcmc_logs/Felidae_preservation_1_mcmc.log

# TPP — additionally plots preservation rates through time
Rscript example_files/plot_preservation_rates.R \
    pyrate_mcmc_logs/Felidae_preservation_1_mcmc.log \
    0.2 \
    example_files/epochs_q.txt
```

The script produces a PDF with four panels:
1. Posterior histogram of mean preservation rate (q)
2. Posterior histogram of Gamma shape (α) — if `-mG` was used
3. MCMC trace of q (to visually check convergence)
4. Preservation rates through time with 95% CI — if TPP epochs are provided

## Impact of preservation on speciation and extinction estimates

Preservation rates and their heterogeneity affect inferred diversification rates in two ways:

### 1. Low preservation inflates apparent extinction

When **q** is small, many lineages will appear to have a shorter fossil record than their true duration. PyRate models this explicitly — but if q is severely underestimated (e.g., from a misspecified model), extinction rates will be overestimated because lineages appear to disappear earlier than they truly did.

**Best practice:** Use the TPP model for datasets spanning >10 My, especially those crossing known rock-record gaps (e.g., Jurassic/Cretaceous boundary, end-Permian).

### 2. Among-lineage heterogeneity biases rate shifts

Without the Gamma correction, a clade containing both well-preserved (e.g., large-bodied, marine) and poorly-preserved (e.g., small, terrestrial) taxa will show **artificial rate shifts** at times when preservation quality changes. The Gamma model absorbs this variation, giving cleaner speciation/extinction rate estimates.

### Running models with and without the Gamma to compare

```bash
# Without Gamma correction
python PyRate.py .../Felidae_preservation_PyRate.py \
    -qShift .../example_files/epochs_q.txt -A 4 \
    -n 5000000 -s 1000

# With Gamma correction
python PyRate.py .../Felidae_preservation_PyRate.py \
    -qShift .../example_files/epochs_q.txt -mG -A 4 \
    -n 5000000 -s 1000
```

Compare the resulting `sp_rates.log` and `ex_rates.log` files using `-plotRJ` (see [Tutorial #3](https://github.com/dsilvestro/PyRate/blob/master/tutorials/pyrate_tutorial_3.md#plot-rates-through-time-and-rate-shifts)). Significant differences in inferred speciation or extinction rate shifts between the two runs indicate that **preservation heterogeneity matters** for your dataset.

## Reproducible example: Felidae dataset

The following example uses a small dataset of 12 Felidae (cat family) lineages spanning the Miocene to Recent (~25–0 Ma). It demonstrates all four preservation models.

### Files provided

| File | Description |
|---|---|
| `example_files/Felidae_preservation_example.txt` | Raw occurrence table (one row per fossil occurrence, with min/max age ranges) |
| `example_files/Felidae_preservation_PyRate.py` | Ready-to-run PyRate input (single replicate, midpoint ages) |
| `example_files/epochs_q.txt` | Epoch boundaries for TPP model (full Phanerozoic) |
| `example_files/plot_preservation_rates.R` | R script to plot and summarize preservation output |

### Step 1 – Inspect the dataset

```bash
python PyRate.py example_files/Felidae_preservation_PyRate.py -data_info
```

This prints a summary of the dataset: number of lineages, total occurrences, time span.

### Step 2 – Run preservation model testing

```bash
python PyRate.py example_files/Felidae_preservation_PyRate.py \
    -qShift example_files/epochs_q.txt \
    -PPmodeltest
```

Read the output to determine whether NHPP, HPP, or TPP best fits the data.

### Step 3 – Run the selected model (example: TPP + Gamma)

Assuming TPP is selected, run the full MCMC:

```bash
python PyRate.py example_files/Felidae_preservation_PyRate.py \
    -qShift example_files/epochs_q.txt \
    -mG \
    -pP 1.5 0 \
    -A 4 \
    -n 5000000 \
    -s 1000
```

For larger datasets or to obtain well-converged estimates, increase `-n` to 10,000,000 or more and run multiple replicates (see below).

**Key flags summary:**

| Flag | Meaning |
|---|---|
| `-qShift epochs_q.txt` | TPP: estimate separate q per epoch |
| `-mG` | Add Gamma heterogeneity across lineages |
| `-pP 1.5 0` | Gamma prior with estimated rate (recommended for TPP) |
| `-A 4` | Use RJMCMC algorithm (recommended) |
| `-n 5000000` | Number of MCMC iterations |
| `-s 1000` | Sampling frequency (record every 1000th state) |

### Step 4 – Check convergence

Open the output `pyrate_mcmc_logs/Felidae_preservation_1_mcmc.log` in [Tracer](https://github.com/beast-dev/tracer/releases). Check that:
- ESS (Effective Sample Size) for `q_rate`, `alpha`, and `posterior` are all > 200.
- Trace plots show good mixing (no trends, no long plateaus).

If ESS is too low, increase `-n` and re-run, or combine runs from multiple replicate datasets.

### Step 5 – Plot preservation rates

```bash
# Using PyRate's built-in TPP plotter
python PyRate.py -plotQ \
    pyrate_mcmc_logs/Felidae_preservation_1_mcmc.log \
    -qShift example_files/epochs_q.txt \
    -b 200

# Using the provided R helper script
Rscript example_files/plot_preservation_rates.R \
    pyrate_mcmc_logs/Felidae_preservation_1_mcmc.log \
    0.2 \
    example_files/epochs_q.txt
```

### Step 6 – Plot speciation and extinction rates through time

```bash
python PyRate.py -plotRJ pyrate_mcmc_logs/ -b 200
```

This generates PDFs with speciation and extinction rate-through-time curves (see [Tutorial #3](https://github.com/dsilvestro/PyRate/blob/master/tutorials/pyrate_tutorial_3.md#plot-rates-through-time-and-rate-shifts)).

### Step 7 – Run multiple replicates and combine

For a publishable analysis, generate 10 replicated datasets in R:

```r
source(".../PyRate/pyrate_utilities.r")
extant_felids <- c("Panthera_leo", "Panthera_tigris", "Acinonyx_jubatus")
extract.ages(file = ".../PyRate/example_files/Felidae_preservation_example.txt",
             extant_species = extant_felids,
             replicates = 10)
```

Then run PyRate on each replicate (`-j 1` through `-j 10`):

```bash
for i in $(seq 1 10); do
  python PyRate.py example_files/Felidae_preservation_example_PyRate.py \
      -qShift example_files/epochs_q.txt -mG -pP 1.5 0 \
      -A 4 -n 5000000 -s 1000 -j $i
done
```

Combine results after all runs:

```bash
python PyRate.py -combLogRJ pyrate_mcmc_logs/ \
    -b 1000 -tag Felidae_preservation_example
```

## Best practices checklist

Use this checklist before submitting a PyRate preservation analysis for publication:

- [ ] **Input data** — Each taxon has ≥3 fossil occurrences. Ages include uncertainty (min/max ranges). Extant taxa are correctly flagged.
- [ ] **Multiple replicates** — Ran ≥3 replicates with independently resampled ages to incorporate stratigraphic uncertainty.
- [ ] **Model testing** — Used `-PPmodeltest` to compare NHPP, HPP, and TPP before the full MCMC run.
- [ ] **Gamma heterogeneity** — Added `-mG` to the winning base model.
- [ ] **Prior sensitivity** — If using TPP, used `-pP 1.5 0` (estimated hyper-prior) rather than fixed rate parameters.
- [ ] **MCMC convergence** — ESS > 200 for all key parameters; checked trace plots in Tracer.
- [ ] **Burn-in** — Discarded at least 10–20% of samples as burn-in (more if convergence was slow).
- [ ] **Results interpretation** — Reported posterior mean and 95% credible interval for q (and α if Gamma model used).
- [ ] **Preservation effect on diversification** — Compared diversification rate estimates with and without `-mG` to assess sensitivity.
- [ ] **Geological context** — Discussed intervals with elevated or depressed preservation rates in terms of rock record completeness, sampling effort, or environmental change.
